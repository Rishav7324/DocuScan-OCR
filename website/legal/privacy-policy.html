1. Who We Are & What This App Does

DocuScan OCR (the "App") is an offline-first document scanning and optical character recognition application for Android. It allows you to capture documents using your device's camera, extract text using on-device machine learning (ML Kit), organize scans into folders, encrypt private documents, and optionally upload copies to cloud storage providers you control.

The App is developed and published by an individual developer. The App is distributed as a compiled Android application.

This Privacy Policy explains what data the App accesses, how that data is processed, where it is stored, and what rights you have over it.

2. What Data We Collect

DocuScan OCR is designed on a principle of local-first, zero-server architecture. The App does not collect, transmit, or sell your personal data. Here is exactly what the App accesses and why:

2.1 Data You Actively Provide

  • Document images — captured via the camera or imported from your device gallery. These are stored locally on your device.
  • OCR text — generated by ML Kit from your document images. Stored locally.
  • Folder names and PINs — folder names you create. PINs are never stored in plaintext (see §6).
  • Cloud sync credentials — R2 access keys, OAuth tokens for Google Drive and Dropbox, stored locally in app-private SharedPreferences.

2.2 Data Accessed via Device Permissions

  • Camera — required to capture document images. No images are recorded without your explicit action (tapping the shutter button).
  • Storage — used to save exported PDF, DOCX, and TXT files to your device, and to import images from the gallery.
  • Biometric hardware — optional. Used only if you choose to unlock private folders with your fingerprint or face unlock instead of entering a PIN.

2.3 Data Not Collected

  • No account information — the App does not have user accounts. No registration, no email, no username.
  • No location data — the App does not access your device's location.
  • No contact data — the App does not access your contacts, call log, or messaging.
  • No advertising ID — the App contains no advertising SDKs and does not access your Google Advertising ID.
  • No payment data — the App is free. There are no in-app purchases, subscriptions, or transactions.
  • No health or biometric data — the App does not collect health information. Face or fingerprint data used for biometric unlock is handled entirely by the Android Biometric API and never reaches the App.

3. We Do Not Operate a Backend Server

DocuScan OCR has no backend infrastructure. The developer does not operate any server that receives data from the App. There is no cloud API endpoint called by the App for its core functionality. There is no user database, no analytics database, and no data processing server.

The only network requests the App makes are:

  • OAuth token exchange — directly between your device and Google's or Dropbox's OAuth endpoints, initiated when you tap Connect in the Cloud Sync screen.
  • Cloud uploads — directly from your device to your chosen cloud provider's API endpoint (Cloudflare R2, Google Drive v3, or Dropbox). The developer has no access to these uploads.
  • ML Kit model download — the first time you use OCR, Google Play Services may download the ML Kit text recognition model. This is a Google service, not controlled by the App.

4. Camera Permission & Image Data

The App requests the CAMERA permission to provide a live viewfinder and capture document images. Here is exactly what happens at each step:

  1. Viewfinder — the camera feed is displayed in a live preview. No image data is recorded or stored during preview.
  2. Capture — when you tap the shutter button, a single JPEG image is captured via CameraX. This image is held in memory and displayed in the batch queue.
  3. Batch finalization — when you confirm the batch, the image is saved to your device's app-internal storage. The image is also processed through ML Kit for OCR.
  4. Encryption — if the document is created inside a private folder, the image is encrypted with AES-256-GCM before being written to disk.

The camera is never accessed in the background. The App does not record video. No camera data is ever transmitted to the developer or any third party (except as described in §7 for cloud sync, which you explicitly configure).

5. Local Storage — Documents, OCR Text, Audit Logs

All document data is stored in the App's private internal storage:

  • SQLite database (Room) — stores folder metadata, document metadata, page references, OCR text, and the audit log. Located at /data/data/com.docuscan.ocr/databases/.
  • Image files — stored in app-internal storage under files/. If the document is in a private folder, the file is encrypted and saved with a .enc extension.
  • SharedPreferences — stores app preferences (theme selection, auto-backup toggle, cloud sync config including OAuth tokens). Located at /data/data/com.docuscan.ocr/shared_prefs/.
  • Export files — PDF, DOCX, and TXT exports are written to external storage under your device's standard documents directory (DocuScan/exports/), which you can access via any file manager or share via the system share sheet.

This data persists until you uninstall the App or clear its app data. Uninstalling the App will remove all local data, including encrypted documents (see §12 for retention details).

6. Encryption & Private Folders

Private folders use a layered encryption model:

  • Android Keystore — a master AES-256-GCM key is generated inside the device's hardware-backed Keystore. This key never leaves secure hardware.
  • PBKDF2 key derivation — your folder PIN is combined with a random salt and stretched through 100,000 iterations of PBKDF2-HMAC-SHA256 to produce a 256-bit file encryption key.
  • AES-GCM file encryption — each page image is encrypted with AES-256 in GCM mode (random 12-byte IV, 128-bit authentication tag) before being written to disk.
  • PIN storage — your PIN is never stored. Only an encrypted verification marker is persisted. If the PIN is forgotten, the encrypted files cannot be recovered.

This encryption is managed entirely on your device. The developer cannot decrypt your files, cannot reset your PIN, and has no access to your encryption keys.

7. Cloud Sync — What Leaves Your Device

Cloud sync is an opt-in feature. You configure it manually in the Cloud Sync screen. When enabled:

  • Cloudflare R2 — document images are uploaded to your R2 bucket using AWS SigV4-signed requests. You supply the endpoint, bucket name, and API credentials.
  • Google Drive — document images are uploaded to your Google Drive via the Drive v3 multipart API. The app requests the drive.file scope, meaning it can only read and write files it created — it cannot see or modify other files in your Drive. You must authorize this via OAuth consent.
  • Dropbox — document images are uploaded to your Dropbox app folder via the files/upload API. You must authorize this via OAuth consent.

The developer does not have access to any cloud provider's credentials or to the uploaded data. Uploads are direct from your device to the provider's endpoint. The developer cannot view, modify, or delete data stored in your cloud accounts through this App.

If you do not configure cloud sync, no data ever leaves your device.

8. OAuth Tokens — Google Drive & Dropbox

When you connect to Google Drive or Dropbox, the App obtains an OAuth 2.0 access token (and a refresh token) from the provider. These tokens are:

  • Stored locally — in app-private SharedPreferences on your device, not on any server.
  • Scoped — Google Drive tokens are limited to the drive.file scope. Dropbox tokens are limited to the app folder.
  • Revocable — you can revoke any token at any time from the provider's security dashboard (Google Account permissions or Dropbox security settings).
  • Not encrypted at rest — tokens are stored as plaintext in SharedPreferences. This is an accepted risk because tokens are scoped and revocable. Encrypting them would require a key stored on the same device, which does not materially improve protection against a device-level attacker.

The developer does not receive, store, or process your OAuth tokens. The token exchange happens directly between your device and the provider's OAuth endpoint.

9. Analytics & Crash Reporting

DocuScan OCR does not include any analytics SDK, crash reporting SDK, or telemetry. The App does not track user behavior, measure feature usage, or report crashes to the developer.

There is no Firebase Analytics, no Google Analytics, no Sentry, no Crashlytics, no Mixpanel — none. If the App crashes, the crash information stays on your device in the Android system log and is not transmitted anywhere.

11. Children's Privacy

DocuScan OCR is a general-purpose document scanning application. It is not directed at children under the age of 13 (or under 16 in the European Union, or under 18 in India under the Digital Personal Data Protection Act 2023). The App does not knowingly collect personal information from children.

Because the App operates without user accounts and does not collect any personal information on a server, there is no registration mechanism that could verify a user's age. If you are a parent or guardian and you believe a child has used the App in a way that concerns you, please contact us at the address in §17. Since the App stores all data locally, the appropriate action is to uninstall the App from the child's device, which removes all local data.

12. Data Retention

Since the developer does not operate any server and does not collect any personal information, data retention is entirely within your control:

  • Local data — documents, OCR text, audit logs, and preferences remain on your device until you delete them within the App or uninstall the App. Uninstalling the App removes all data stored in app-internal storage.
  • Export files — PDF, DOCX, and TXT files you create via the Export feature are saved to your device's external storage (Documents/DocuScan/exports/) and persist after the App is uninstalled. You must delete these files manually or via a file manager.
  • Cloud copies — if you enabled cloud sync, copies of your document images exist in your chosen cloud provider's storage until you delete them from that provider. The App cannot delete them on your behalf.

The developer does not store any backup, archive, or cache of your data. When the App is uninstalled, the developer has no record that you ever used it.

13. Your Privacy Rights (GDPR, CCPA, DPDP, Others)

Depending on where you live, you may have legal rights over your personal data. Because DocuScan OCR operates without a backend server and does not collect, store, or process personal data on the developer's systems, most of these rights are exercised by actions you take within the App itself or on your device.

13.1 Data Subject Rights (GDPR — EU/EEA/UK)

If you are in the European Union, European Economic Area, or the United Kingdom, the General Data Protection Regulation (GDPR) grants you the following rights. Here is how each applies to DocuScan OCR:

  • Right of access — you can access all your data by navigating through the App's Dashboard and Document screens. The local SQLite database is not directly accessible but all its content is displayed in the UI.
  • Right to rectification — you can edit document names and reorder pages within the App.
  • Right to erasure ("right to be forgotten") — you can delete individual documents, folders, or the entire database by clearing app data or uninstalling the App.
  • Right to restriction of processing — you can disable cloud sync at any time from the Cloud Sync screen.
  • Right to data portability — you can export any document as PDF, DOCX, or TXT and transfer those files to any other system.
  • Right to object — since the App does not perform profiling, automated decision-making, or direct marketing, there is no processing to object to beyond not using the cloud sync feature.
  • Rights related to automated decision-making — the App does not use automated decision-making that produces legal effects concerning you.

If you believe your data has been processed in violation of the GDPR and you cannot resolve it through the actions above, you have the right to lodge a complaint with your local data protection authority.

13.2 California Privacy Rights (CCPA/CPRA)

If you are a resident of California, the California Consumer Privacy Act (CCPA), as amended by the California Privacy Rights Act (CPRA), grants you similar rights. Because the App does not:

  • Collect personal information (as defined by the CCPA) on a server,
  • Sell or share personal information for cross-context behavioral advertising,
  • Use sensitive personal information for purposes beyond those necessary to provide the service,

the primary CCPA rights (right to know, right to delete, right to opt out of sale/sharing) are exercised through the same on-device actions described above. The App does not have a "Do Not Sell or Share My Personal Information" link because no such activity occurs.

13.3 India — Digital Personal Data Protection Act, 2023 (DPDP Act)

If you are in India, the DPDP Act grants data principals (you) rights of access, correction, erasure, and grievance redressal. As noted throughout this policy, the App operates without a server and does not collect personal data on the developer's systems. You can exercise your rights directly within the App. If you have any grievance, please contact us using the details in §17.

13.4 Other Jurisdictions

If your country has enacted a privacy law that grants you additional rights not described above (such as Brazil's LGPD, Canada's PIPEDA, Australia's Privacy Act, or similar), the same principle applies: your data is under your control on your device, and the App's local-first architecture means you can exercise your rights by managing data directly within the App. Please contact us if you have specific questions about how your local law applies.

14. International Data Transfers

DocuScan OCR does not transfer your personal data to any country. The App has no backend server. Your data stays on your device and optionally in your chosen cloud provider's storage.

If you choose to use cloud sync, your document images are transferred to the cloud provider's servers. The location of those servers depends on the provider's infrastructure and your account configuration:

  • Cloudflare R2 — you select the region when creating your bucket. See Cloudflare's data protection documentation.
  • Google Drive — data is stored in Google's global infrastructure. See Google's Privacy Policy.
  • Dropbox — data is stored in Dropbox's infrastructure. See Dropbox's Privacy Policy.

The developer does not control where these providers store data. You should review each provider's policies before enabling sync.

15. Security Measures

The App implements the following security measures to protect your data:

  • Full-disk encryption — on devices with Android 10+ (API 29+), the entire /data partition is encrypted by the operating system. This provides a baseline level of protection for all app data at rest.
  • Android Keystore — an AES-256-GCM master key is stored in hardware-backed Keystore on supported devices (API 28+). The key material cannot be extracted from the secure processor.
  • AES-256-GCM file encryption — images in private folders are encrypted with AES-256 in Galois/Counter Mode, which provides both confidentiality and integrity. Tampering with encrypted files is detectable on decryption.
  • PBKDF2-HMAC-SHA256 key derivation — folder PINs are stretched through 100,000 iterations to derive encryption keys. This rate-limits brute-force attacks against the PIN.
  • In-memory PIN — the folder PIN is held only in memory (Kotlin StateFlow) and is cleared when the App goes to the background.
  • PKCE OAuth — all OAuth flows use Proof Key for Code Exchange, preventing authorization code interception attacks.
  • HTTPS/TLS — all network requests to cloud providers use HTTPS with TLS encryption.
  • Minimal attack surface — the App uses only 7 runtime dependencies and does not include any analytics, crash reporting, or advertising SDKs.

For a detailed threat model analysis covering specific attack vectors and their mitigations, see the Security & Responsible Disclosure Policy.
